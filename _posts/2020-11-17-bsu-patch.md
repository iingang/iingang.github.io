# BSU Patch

---





`WLS 12.1.2 이하 버전`에서 패치 시, **bsu** 를 이용해 패치함.



패치 전, bsu 자체를 update 해 패치 시간을 단축시킬 수 있음.

bsu V4 가 현재 릴리즈 된 가장 최신 버전

![image-20201117193133828](/assets/img/posts/bsu_update.png){: width="400" class="normal"}

<br/>

<br/>

## <mark style='background-color: #e8fffc'>bsu update 방법</mark>

---

1. **$ cd {MW_HOME}/utils/bsu**
2. 위의 31136426 패치 파일을 해당 경로에 압축 해제
3. **`. ./bsu_update.sh install`** **(리눅스)**  /  **`bsu_update.bat install`** **(윈도우)**

<br/>

<br/>

## <mark style='background-color: #e8fffc'>bsu Patch 적용 순서</mark>

---

**WLS 10.3.6 PSU201020** 패치를 적용해보자!

> 리눅스 기준으로 작성됨

<br/>



1. **웹로직 기동 계정으로 로그인**

   - su - weblogic

     <br/>

2. **엔진 백업**

   - 도메인홈 제외한 엔진단 디렉토리들 tar로 압축

     <br/>

3. **웹로직 Admin Server 및 Managed Server 중지**

   - ex) ./stopA.sh 

   - ex) ./stopM1.sh

     <br/>

4. **bsu.sh 의 힙 사이즈 수정 필요** (최소 2기가 이상)

   - **`MEM_ARGS = "-Xms4096m -Xmx4096m"`**

     <br/>

5. **기존 패치 리스트 확인**

   - **`./bsu.sh prod_dir=${MW_HOME}/wlserver_10.3 -status=applied -view`**

     <br/>

6. **패치 목록이 존재하면 해당 패치 롤백**

   - **`./bsu.sh -prod_dir=${MW_HOME}/wlserver_10.3 -remove -patchlist=[PATCH_ID] -verbose`**

   - PATCH_ID는 패치파일 내의 {PATCH_ID}.jar 파일로 확인 가능

     <br/>

7. **패치 적용**

   - **패치 파일의 default 위치는 ${MW_HOME}/utils/bsu/cache_dir**

   - 위의 경로에서 패치 파일 압축 해제 (p31641257_1036_Generic.zip)

   - {MW_HOME}/utils/bsu 위치에서 패치 명령어 실행

     - **`./bsu.sh -prod_dir=${MW_HOME}/wlserver_10.3 -install -patchlist=NA7A -verbose`**
     - 에러 발생 시, 로그 확인을 원하면 <mark style="background-color: #fff5b1">-log=bsu.log -log_priority=trace</mark> 옵션을 붙이면 해당 경로에 생성된 로그를 볼 수 있음

   - 패치 성공이 뜨면, 패치 리스트 정상적으로 적용됐는지 확인할 것

     <br/>

8. **웹로직 Admin Server 및 Managed Server 기동**

   - ex) ./startA.sh 

   - ex) ./startM1.sh

     <br/>
